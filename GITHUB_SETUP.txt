# GITHUB SETUP — Do this once, takes about 20 minutes

## Step 1 — Create GitHub account
https://github.com/signup
Free account is all you need.

## Step 2 — Create a new private repo
- Go to https://github.com/new
- Name it: ghost-arbitrage-scanner
- Set to PRIVATE
- Click "Create repository"

## Step 3 — Upload the files
- Click "uploading an existing file" on the repo page
- Download the ghost_arbitrage_scanner folder from your Google Drive
- Drag the entire folder contents into GitHub
- Commit with message: "initial build"

Folder structure GitHub needs to see:
  .github/workflows/daily_scan.yml
  .github/workflows/weekly_analysis.yml
  config/keywords.yaml
  config/markets.yaml
  config/scoring.yaml
  config/sources.yaml
  src/main.py
  src/models.py
  src/storage.py
  src/normalizer.py
  src/scorer.py
  src/deduper.py
  src/digest.py
  src/weekly_analysis.py
  src/collectors/govdeals.py
  src/collectors/rss.py
  requirements.txt
  README.md

## Step 4 — Add your secrets (replaces the .env file)
- Go to your repo → Settings → Secrets and variables → Actions
- Click "New repository secret" for each of these 3:

  Name: DIGEST_TO_EMAIL
  Value: your personal email (where you want reports sent)

  Name: DIGEST_FROM_EMAIL
  Value: a Gmail address you control (can be the same)

  Name: GMAIL_APP_PASSWORD
  Value: your Gmail App Password (see below)

### Getting a Gmail App Password:
1. Go to https://myaccount.google.com/security
2. Enable 2-Step Verification if not already on
3. Go to https://myaccount.google.com/apppasswords
4. Create app: name it "Ghost Scanner"
5. Copy the 16-character password — paste it as the secret above

## Step 5 — Test it manually
- Go to your repo → Actions tab
- Click "Daily Ghost Scanner" workflow
- Click "Run workflow" → Run
- Watch it execute in real time
- Check your email in ~3 minutes

## Step 6 — You're done
After the manual test works, it runs automatically:
- Every day at 7:00 AM Eastern — daily digest email
- Every Sunday at 8:00 AM Eastern — weekly analysis email

## What the weekly email tells you:
- CONTINUE — system is finding real opportunities, keep running
- NEEDS TUNING — low signal, adjust keywords or scoring
- KILL — nothing worth pursuing, reassess the model

## Expanding to a new market later:
1. Open config/markets.yaml
2. Change active: false to active: true for the market you want
3. Commit the change
4. GitHub picks it up automatically next run
